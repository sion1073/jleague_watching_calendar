クリーンアップしても状況は変わりません。動作確認時の試合リストの昇順/降順は昇順に設定しています。

データを投入し動作確認しましたが、仕様通りにソートされていない可能性があります。

## 回答

*修正後のコードが正しく実装されているか確認*

*jleague_data_20260309_212544.json を確認して実際のデータ構造を分析*

コードは正しく修正されていますが、データ内のmatchDateに時刻情報が含まれていることが問題です。具体的には：

- `2026-02-21T09:20:23.062` - FC東京 vs 川崎フロンターレ
- `2026-02-21T00:00:00.000` - 鹿島アントラーズ vs 柏レイソル

DateTime.compareTo() は時刻も比較するため、同じ日付でも時刻が異なると、チーム名でのソートより時刻での順序が優先されてしまいます。これが原因でユーザーが報告した順序になっていました。

**原因**: matchDate から年月日のみを抽出して比較するべき

*season_detail_screen.dart のソート処理を修正*

*team_detail_screen.dart のソート処理を修正*

修正されたコード：

```dart
..sort((a, b) {
  // 日付部分のみ（年月日）で比較
  final aDate = DateTime(a.matchDate.year, a.matchDate.month, a.matchDate.day);
  final bDate = DateTime(b.matchDate.year, b.matchDate.month, b.matchDate.day);

  if (settings.matchSortAscending) {
    final dateComparison = aDate.compareTo(bDate);
    if (dateComparison != 0) return dateComparison;
    return a.homeTeam.compareTo(b.homeTeam);
  } else {
    final dateComparison = bDate.compareTo(aDate);
    if (dateComparison != 0) return dateComparison;
    return b.homeTeam.compareTo(a.homeTeam);
  }
});
```

年月日のみを抽出することで、時刻情報の影響を排除し、同日の試合が正しくチーム名でソートされるようになります。
