# 議事録フォーマットガイド

このドキュメントは、AI開発セッションの議事録を作成する際の標準フォーマットを定義します。

---

## 基本構造

```markdown
# [タイトル] 議事録

**日付**: YYYY-MM-DD

---

## 質問

ユーザーの質問や発言をそのまま見出しに

---

## 回答

AIの回答内容をここに記載。

ツールの使用は *説明* の形式でイタリック体で記載。

---

## 質問

次のユーザーの質問や発言

---

## 回答

次の回答内容...

---
```

---

## 重要なルール

### 1. ユーザーの発言を見出し（##）にそのまま使用しない

❌ 間違い:
```markdown
## ユーザーの質問内容をそのまま見出しに
```

✅ 正しい:
```markdown
## 質問
ユーザーの質問内容
```

### 2. 見出しの後は必ず区切り線（---）

```markdown
## ユーザーの発言

---

## 回答
```

### 3. AIの回答は必ず「## 回答」見出し

```markdown
## 回答

回答の内容をここに記載します。
```

### 4. ツールの使用は *イタリック体* で簡潔に

```markdown
*lib/services/auth_service.dart を作成*

*git status を確認*

*README.mdを更新*
```

### 5. 各やり取りの間は区切り線（---）で区切る

## 実例

### 例1: 単一の質問と回答

```markdown

## 回答

データモデルの設計について説明します。

以下のような構造をお勧めします：

[詳細な説明]

---
```

## 回答

*lib/services/auth_service.dart を作成*

READMEからも削除します。

*README.md を更新*

完了しました。

---
```

## セクション構造

### 1. ヘッダー

```markdown
# [タイトル] 議事録

**日付**: YYYY-MM-DD

---
```

### 2. やり取りセクション（繰り返し）

```markdown
## 質問

ユーザーの発言

---

## 回答

回答内容

---
```

### 3. 回答内の要素

回答には以下を含めることができます：

- **テキストによる説明**
- **コードブロック** (```で囲む)
- **箇条書き** (- や 1. を使用)
- **見出し** (### や #### を使用)
- **ツール使用の記録** (*イタリック体*)
- **強調** (**太字** や *イタリック*)

---

## よくある間違いと修正

### 間違い1: 質問を見出しに利用する

❌ 間違い:
```markdown
## 認証機能を実装してください
```

✅ 正しい:
```markdown
## 質問
認証機能を実装してください
```

### 間違い2: 区切り線を忘れる

❌ 間違い:
```markdown
## ユーザーの発言
## 回答
```

✅ 正しい:
```markdown
## ユーザーの発言

---

## 回答
```

### 間違い3: ツール使用を詳細に書きすぎる

❌ 間違い:
```markdown
Readツールを使用して、/path/to/file.dart を読み込み、内容を確認しました。
```

✅ 正しい:
```markdown
*file.dart を確認*
```

### 間違い4: 回答を要約する

❌ 間違い（要約形式）:
```markdown
## 回答

認証機能を実装しました。
```

✅ 正しい（文言そのまま）:
```markdown
## 回答

議事録を確認して、次フェーズの「簡易パスワード認証の実装」を進めます。まず議事録の内容を確認させてください。

*議事録を確認*

議事録を確認しました。次フェーズは「簡易パスワード認証の実装」ですね。タスクを計画して実装を進めます。

[実際の回答全文...]
```

## チェックリスト

議事録作成時に以下を確認：

- [ ] ユーザーの発言は「## 質問」見出しで始まる
- [ ] 各見出しの後に区切り線（---）がある
- [ ] AIの回答は「## 回答」見出しで始まる
- [ ] ツール使用は *イタリック体* で簡潔に記載
- [ ] やり取りの間に区切り線（---）がある
- [ ] 文言がそのまま記録されている（要約していない）
- [ ] コードブロックは ``` で適切に囲まれている
- [ ] 日付が正しく記載されている

## ファイル命名規則

```
YYYY-MM-DD_NN.md
```

- `YYYY`: 年（4桁）
- `MM`: 月（2桁）
- `DD`: 日（2桁）
- `NN`: その日の連番（01, 02, ...）

例：
- `2026-02-04_01.md` - 2026年2月4日の1つ目のセッション
- `2026-02-04_02.md` - 2026年2月4日の2つ目のセッション
